# Community

HyperSentry Labs is built in the open, and the community around it is part
of what makes that work.

## Where the community lives

- **Discussions** — [github.com/orgs/hypersentry-labs/discussions](https://github.com/orgs/hypersentry-labs/discussions)
  is the primary hub for questions, ideas, and sharing what you've built.
- **Issues** — Each repository's issue tracker is used for bugs, feature
  requests, and documentation problems specific to that project.
- **Pull Requests** — Code and documentation contributions happen through
  pull requests on individual repositories.

## How to get involved

- **Use the software** and report what doesn't work. Well-written bug
  reports are one of the most valuable contributions.
- **Improve documentation.** Clear docs help everyone, and documentation
  fixes are one of the easiest ways to make a first contribution.
- **Answer questions** in Discussions or on open issues. Helping other
  users is as valuable as writing code.
- **Contribute code.** See [CONTRIBUTING.md](CONTRIBUTING.md) for the
  workflow and expectations.
- **Share what you build.** Use the **Show & Tell** discussion category to
  showcase projects built with HyperSentry Labs tools.

## Expectations

All community spaces are governed by the
[Code of Conduct](CODE_OF_CONDUCT.md). We expect everyone — maintainers and
contributors alike — to engage respectfully and in good faith.

## Recognition

Contributions are recognized through:

- Attribution in release notes for merged pull requests
- Listing in a repository's contributor graph
- Consideration for maintainer status for sustained, high-quality
  contributions, per [GOVERNANCE.md](GOVERNANCE.md)

## Staying up to date

Watch individual repositories, or follow the
[organization profile](https://github.com/hypersentry-labs) for an overview
of active projects.
