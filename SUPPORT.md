# Support

This document explains where to get help with HyperSentry Labs projects.

## Before asking for help

- Check the project's `README.md` for setup and usage instructions.
- Search existing [issues](https://github.com/orgs/hypersentry-labs/repositories)
  and [discussions](https://github.com/orgs/hypersentry-labs/discussions) —
  your question may already be answered.
- Review the project's documentation, if it maintains a dedicated docs site
  or `docs/` directory.

## Where to get help

| Need                       | Channel                                                                                |
| -------------------------- | -------------------------------------------------------------------------------------- |
| Usage question             | [Q&A Discussions](https://github.com/orgs/hypersentry-labs/discussions/categories/q-a) |
| Bug report                 | The project's **Bug Report** issue form                                                |
| Feature request            | The project's **Feature Request** issue form                                           |
| Documentation problem      | The project's **Documentation** issue form                                             |
| Security vulnerability     | See [SECURITY.md](SECURITY.md) — do not open a public issue                            |
| General discussion or idea | [General or Ideas Discussions](https://github.com/orgs/hypersentry-labs/discussions)   |

## Response expectations

HyperSentry Labs projects are maintained on a volunteer basis. There is no
guaranteed response time, though most issues and discussions receive an
initial response within a reasonable timeframe. Well-documented issues with
clear reproduction steps are easier to act on quickly.

## Commercial support

HyperSentry Labs does not currently offer paid or commercial support
contracts for any project.
